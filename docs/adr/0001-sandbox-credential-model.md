# Use split credentials for Docker Sandbox Pi

We use Docker custom secrets with format-valid placeholders for Vercel and Supabase so their CLIs can pass local token validation while the real personal tokens remain on the host and are injected by Docker's proxy. Pi ChatGPT/Codex OAuth cannot use Docker's OpenAI proxy secret because Pi decodes OAuth state locally, so the sandbox wrapper mounts a dedicated read-write `~/.cache/wedding-planner/pi-auth` cache and symlinks it to Pi's `auth.json`; this intentionally exposes the main ChatGPT OAuth state to sandbox processes to avoid logging in for every new sandbox. The sandbox auto-links the documented Vercel/Supabase production targets and does not add command guardrails, favoring smooth full-project operations over blast-radius reduction.
